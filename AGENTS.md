# AGENTS.md

이 문서는 이 프로젝트에서 AI 도구가 어떤 기준으로 작업해야 하는지 정리한 안내서입니다.

## 목표

이 프로젝트의 목표는 비개발자도 이해할 수 있는 AI 자동화 컨설팅 제안서 PPT를 만드는 것입니다.

최종 제안서는 내부 개발팀이 없는 10-50명 규모의 SMB 대표가 바로 이해할 수 있어야 합니다.

제안서에서 설명해야 할 핵심은 다음과 같습니다.

- 현재 업무에서 반복되는 일을 찾기
- AI와 자동화로 줄일 수 있는 시간을 보여주기
- 적용 가능한 자동화 아이디어를 제안하기
- 4주 컨설팅 진행 방식과 기대 효과를 정리하기

## 말투

- 쉬운 우리말로 설명합니다.
- 어려운 기술 용어는 첫 등장 때 한 문장으로 풀어 설명합니다.
- 대표가 바로 이해할 수 있게 과장보다 구체적인 효과를 우선합니다.
- 기능 설명보다 고객의 업무가 어떻게 편해지는지를 먼저 말합니다.
- "AI가 모든 일을 대신한다"보다 "반복되는 일을 덜 하게 만든다"는 표현을 우선합니다.

## 문서 작성 기준

- 한 문장은 가능한 짧게 씁니다.
- 한 문단에는 하나의 메시지만 담습니다.
- 고객이 얻는 효과를 중심으로 설명합니다.
- 필요한 경우 쉬운 예시를 함께 씁니다.
- 특정 브랜드의 공식 디자인 시스템이라고 표현하지 않습니다.

## PPT 제작 규칙

- PPT 제작이 필요할 때는 Presentations 플러그인을 우선 활용합니다.
- 텍스트, 차트, 기본 도형은 가능하면 PowerPoint에서 편집 가능한 객체로 유지합니다.
- 중요한 글자는 이미지 안에 넣지 말고 PPT 텍스트로 배치합니다.
- [brand/DESIGN.md](brand/DESIGN.md)의 색상, 배경, 버튼, 여백 규칙을 우선 적용합니다.
- 제안서는 장식보다 이해하기 쉬운 구조를 우선합니다.
- 완료 전 `output/review-montage.png`와 검수 요약을 만듭니다.

## 이미지 활용 규칙

- 이미지가 필요한 영역은 `imagegen` 스킬을 활용해 생성할 수 있습니다.
- 전체 덱에서는 최소 3개 정도의 이미지를 활용하는 것을 권장합니다.
- 이미지는 장식용보다 이해를 돕는 용도로 사용합니다.
- 중요한 문장, 숫자, 표, 차트는 이미지 안에 넣지 않고 PowerPoint 텍스트나 편집 가능한 객체로 배치합니다.
- 특정 브랜드의 로고, 제품, 매장, 앱 화면, 멤버십 이미지는 사용하지 않습니다.

## imagegen 저장 확인 절차

이미지를 PPT에 넣기 전에 반드시 아래 순서를 따릅니다.

1. `imagegen`으로 필요한 이미지를 생성합니다.
2. 생성된 이미지 결과 파일을 `assets/` 폴더에 저장하거나 복사합니다.
3. `assets/` 폴더에 실제 이미지 파일이 존재하는지 확인합니다.
4. 파일 크기가 0이 아닌지 확인합니다.
5. 저장이 확인된 이미지 파일만 PPT에 삽입합니다.
6. 저장 확인이 되지 않으면 PPT에 삽입하지 않고, 사용자에게 이미지 파일 저장이 필요하다고 알립니다.

권장 이미지 파일 형식은 다음과 같습니다.

- `.png`
- `.jpg`
- `.webp`

권장 이미지 크기는 16:9 비율입니다.

- `1536x864`
- `1920x1080`

권장 이미지 영역은 다음과 같습니다.

- 표지 또는 도입부에 사용할 추상적인 업무 자동화 이미지
- 반복 업무가 정리되는 모습을 보여주는 업무 흐름 이미지
- 4주 컨설팅 진행 과정을 보조하는 워크숍 또는 협업 분위기 이미지

## 산출물 관리

- 참고 자료와 이미지 후보는 `assets/` 폴더에 둡니다.
- 완성된 PPT, PDF, 이미지 파일은 `output/` 폴더에 둡니다.
- 사업 배경과 제안 방향은 `context/` 폴더에 둡니다.
- 디자인 방향은 `brand/` 폴더에 둡니다.

## 안전 규칙

- 외부 웹 접속이나 데스크톱 앱 조작이 필요하면 먼저 왜 필요한지 설명하고 승인을 요청합니다.
- 비밀키, 결제, 계정 설정 화면은 자동 조작하지 않습니다.
- 사용자의 계정 정보나 민감한 설정을 임의로 변경하지 않습니다.
- 확실하지 않은 내용은 단정하지 말고 확인이 필요한 항목으로 표시합니다.

## 다음 단계 원칙

사용자가 문서 내용을 확인한 뒤, 필요한 수정 사항을 반영합니다.

사용자가 명시적으로 요청하기 전에는 PPT 제작으로 넘어가지 않습니다.

## 작업 완료 후 처리 규칙

- 작업이 완료되면 변경된 파일을 확인하고 Git Commit을 만듭니다.
- Git Push는 사용자에게 먼저 확인을 받은 뒤 진행합니다.
- 사용자가 Push를 승인하지 않으면 로컬 커밋 상태로 멈춥니다.
