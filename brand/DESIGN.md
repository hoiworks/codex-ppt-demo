# 디자인 방향 초안

이 문서는 AI 자동화 컨설팅 제안서 PPT를 만들 때 사용할 디자인 방향을 정리합니다.

특정 브랜드의 공식 디자인 시스템이 아닙니다. 따뜻한 카페 분위기, 깊은 그린 컬러, 여유 있는 여백, 둥근 버튼과 카드 느낌을 참고한 제안서용 스타일 가이드입니다.

## 1. 전체 분위기

제안서는 신뢰감 있고 편안한 인상을 주는 방향으로 만듭니다.

- 차분하고 따뜻한 느낌
- 고급스럽지만 과하게 꾸미지 않은 화면
- 복잡한 기술보다 업무 개선 효과가 먼저 보이는 구성
- 발표자가 설명하기 쉽고, 듣는 사람이 빠르게 이해할 수 있는 흐름

핵심 이미지는 "전문적인 컨설팅 문서이지만 딱딱하지 않은 제안서"입니다.

## 2. 색상 방향

전체 색상은 따뜻한 크림색과 깊은 그린색을 중심으로 사용합니다.

### 기본 색상

- 배경: 따뜻한 크림색 또는 아주 밝은 회색
- 본문 글자: 진한 회색
- 제목 강조: 깊은 그린색
- 버튼과 핵심 강조: 선명한 그린색
- 보조 강조: 연한 민트색 또는 부드러운 베이지색

### 추천 색상 예시

| 역할 | 색상 예시 | 사용 위치 |
|---|---:|---|
| 깊은 그린 | `#1E3932` | 표지, 섹션 구분, 핵심 메시지 영역 |
| 기본 그린 | `#006241` | 큰 제목, 핵심 문장, 강조 라벨 |
| 선명한 그린 | `#00754A` | 버튼, 아이콘, 중요한 숫자 |
| 연한 민트 | `#D4E9E2` | 보조 배경, 흐름도 박스 |
| 따뜻한 크림 | `#F2F0EB` | 전체 배경, 여백이 큰 페이지 |
| 흰색 | `#FFFFFF` | 카드, 표, 본문 영역 |
| 진한 회색 | `#222222` | 본문 텍스트 |
| 보조 회색 | `#666666` | 설명, 주석, 작은 텍스트 |

색상은 많이 쓰지 않습니다. 한 슬라이드 안에서는 배경색 1개, 주요 강조색 1개, 보조색 1개 정도로 제한합니다.

## 3. 글자 스타일

한글 제안서에서는 읽기 쉬운 산세리프 글꼴을 사용합니다.

### 추천 글꼴

- 기본: Pretendard, Noto Sans KR, 맑은 고딕
- 영문과 숫자: Inter, Arial, Pretendard

### 글자 크기 기준

| 역할 | 권장 크기 | 설명 |
|---|---:|---|
| 표지 제목 | 34-44pt | 제안서 제목 |
| 섹션 제목 | 28-34pt | 큰 장 구분 |
| 슬라이드 제목 | 22-28pt | 각 슬라이드의 핵심 문장 |
| 본문 | 15-18pt | 설명 문장 |
| 주석 | 11-13pt | 출처, 보조 설명 |
| 숫자 강조 | 28-40pt | 시간 절감, 비용 절감 등 |

### 문장 작성 기준

- 제목은 한 줄로 끝나는 것이 좋습니다.
- 본문은 긴 문단보다 짧은 문장과 목록을 사용합니다.
- 한 슬라이드에는 핵심 메시지 하나만 담습니다.
- 기술 용어는 필요한 경우에만 쓰고, 바로 쉬운 설명을 붙입니다.

## 4. 슬라이드 구성 원칙

### 기본 레이아웃

- 넓은 여백을 둡니다.
- 중요한 내용은 중앙 또는 왼쪽 상단에 배치합니다.
- 표와 도식은 줄 간격을 넉넉하게 둡니다.
- 카드형 박스는 둥근 모서리와 약한 그림자만 사용합니다.

### 추천 화면 구조

- 표지: 깊은 그린 배경 + 짧은 제목 + 작은 부제
- 문제 제기: 크림 배경 + 큰 문장 + 3개 이하의 문제 카드
- 현재 업무 흐름: 단계형 도식
- 자동화 후 흐름: 이전 흐름과 비교되는 간단한 도식
- 기대 효과: 숫자 중심 카드 3개
- 진행 방식: 3-5단계 타임라인
- 다음 단계: 짧은 체크리스트

## 5. 카드와 박스 스타일

카드는 정보를 묶어서 보여줄 때만 사용합니다.

- 배경: 흰색
- 모서리: 8-12px 정도의 둥근 모서리
- 그림자: 아주 약하게 사용
- 안쪽 여백: 충분히 넓게 사용
- 카드 안에는 제목, 짧은 설명, 숫자 또는 아이콘을 넣습니다.

카드를 너무 많이 쓰면 화면이 복잡해집니다. 한 슬라이드에는 3개 정도가 적당합니다.

## 6. 버튼과 강조 요소

PPT 안에서 실제 버튼처럼 보이는 요소는 "다음 단계", "제안 범위", "핵심 효과" 같은 행동이나 결론을 강조할 때 사용합니다.

- 형태: 둥근 알약형 버튼
- 배경: 선명한 그린
- 글자: 흰색
- 사용 위치: 표지, 마지막 슬라이드, 핵심 제안 슬라이드

버튼처럼 보이는 요소를 너무 많이 쓰지 않습니다. 중요한 행동이나 결론에만 사용합니다.

## 7. 도식과 아이콘

아이콘과 도식은 내용을 쉽게 이해하도록 돕는 보조 요소입니다.

### 추천 도식

- 현재 업무 흐름도
- 자동화 후 업무 흐름도
- 문제에서 해결까지의 단계
- 컨설팅 진행 일정
- 기대 효과 비교표

### 아이콘 사용 기준

- 의미가 바로 이해되는 아이콘만 사용합니다.
- 아이콘 색상은 그린 또는 회색으로 제한합니다.
- 장식용 아이콘은 줄이고, 설명에 도움이 되는 아이콘만 사용합니다.

## 8. 이미지 사용 기준

이 제안서는 제품 홍보물이 아니라 컨설팅 제안서입니다.

따라서 이미지는 분위기용으로 많이 쓰기보다, 업무 상황을 이해시키는 데 필요한 경우에만 사용합니다.

사용 가능한 이미지 예시는 다음과 같습니다.

- 회의 장면
- 업무 자동화 흐름을 보여주는 추상 이미지
- 문서, 데이터, 대시보드 느낌의 화면
- 상담 또는 워크숍 분위기의 사진

특정 브랜드의 로고, 매장, 제품, 멤버십, 앱 화면을 예시로 사용하지 않습니다.

## 9. 피해야 할 것

- 특정 브랜드의 로고나 제품을 넣는 것
- 특정 브랜드의 공식 디자인 시스템이라고 표현하는 것
- 한 슬라이드에 너무 많은 글을 넣는 것
- 장식 이미지를 많이 넣어 핵심이 흐려지는 것
- 너무 많은 색상을 섞는 것
- 기술 설명만 길게 이어지는 것
- 효과를 과장해서 표현하는 것

## 10. 슬라이드별 적용 예시

### 표지

- 깊은 그린 배경
- 큰 흰색 제목
- 작은 크림색 부제
- 오른쪽 아래에 짧은 제안 날짜 또는 회사명

### 문제 요약 슬라이드

- 크림색 배경
- 상단에 한 줄 핵심 문장
- 아래에 문제 카드 3개
- 각 카드에는 아이콘, 문제명, 짧은 설명

### 자동화 제안 슬라이드

- 왼쪽: 현재 업무 흐름
- 오른쪽: 자동화 후 업무 흐름
- 가운데에 화살표 또는 전환 표시
- 개선되는 부분은 선명한 그린으로 강조

### 기대 효과 슬라이드

- 큰 숫자 3개를 중심으로 구성
- 예: 업무 시간 절감, 문서 작성 시간 단축, 반복 입력 감소
- 숫자 아래에는 짧은 설명만 배치

### 다음 단계 슬라이드

- 깊은 그린 배경
- 흰색 체크리스트
- 미팅, 업무 진단, 자동화 후보 선정, PoC 진행 순서 표시

## 11. 최종 방향 요약

이 제안서의 디자인은 따뜻한 크림색 배경과 깊은 그린색 강조를 중심으로 합니다.

읽기 쉽고, 차분하며, 실무적인 컨설팅 문서처럼 보여야 합니다. 핵심은 멋진 장식보다 고객이 "우리 업무에 바로 적용할 수 있겠다"고 느끼게 만드는 것입니다.
